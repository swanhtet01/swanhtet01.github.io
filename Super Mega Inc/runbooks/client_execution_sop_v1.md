# SuperMega Client Execution SOP (V1)

## Purpose

This SOP is the repeatable operating system for every client rollout, from first call to stable AI-native ERP operations.

Use this for:
- Yangon Tyre (pilot)
- future single-tenant clients
- multi-client agency delivery

## Delivery Promise

For each client, SuperMega delivers:
- one live control cockpit for `CEO`, `Director`, and `Manager`
- one structured input layer (sheets/forms) for clean operational updates
- one evidence-linked action loop (owner + due date + closure proof)
- one weekly executive brief with prioritized decisions

## Standard Phases

## Phase 0: Qualification (Day 0-2)

Inputs:
- discovery call
- qualification checklist
- sample files/emails/screenshots

Outputs:
- `go/no-go` decision
- selected package (`Starter`, `Growth`, `Scale`)
- signed scope + success metrics

Exit criteria:
- named sponsor
- named operator from client side
- data access path confirmed

## Phase 1: Data Mapping (Day 2-4)

Inputs:
- Drive folder(s)
- mailbox access
- current trackers/sheets

Outputs:
- source map (`files`, `emails`, `external`, `manual inputs`)
- data quality gaps list
- role map (`CEO`, `Director`, `Manager`, owners)

Exit criteria:
- minimum data coverage score target defined
- field-level schema locked for intake sheets

## Phase 2: Foundation Setup (Day 4-7)

Inputs:
- approved schema
- credentials and access

Outputs:
- Input Center templates deployed
- ERP activity tracking enabled
- DQMS starter registers enabled (if in scope)
- dashboard baseline generated

Exit criteria:
- one end-to-end dry run is successful
- at least one live row in each required intake sheet

## Phase 3: Agent Workflow Activation (Week 2)

Inputs:
- baseline pipeline

Outputs:
- role-based control towers
- automated digest/brief cadence
- escalation rules for high-priority items

Exit criteria:
- decisions are evidence-linked
- no high-priority item without owner + due date

## Phase 4: Adoption Sprint (Week 3-4)

Inputs:
- live usage feedback

Outputs:
- UI simplification pass
- SOP handover docs
- manager/director playbooks

Exit criteria:
- client team runs daily loop without SuperMega intervention
- weekly review cadence is stable

## Phase 5: Commercial Expansion (Month 2+)

Outputs:
- add-on modules enabled
- KPI baseline vs. post-implementation comparison
- upsell path to `Growth` or `Scale`

Exit criteria:
- measurable business outcome achieved
- expansion scope approved

## Standard Cadence

Daily:
- intake sync
- incident/watchlist triage
- owner/date closure tracking

Weekly:
- executive brief
- CAPA and supplier trend review
- backlog cleanup + SLA audit

Monthly:
- module ROI review
- expansion planning

## Roles and Responsibilities

SuperMega:
- Solution Lead: architecture, roadmap, quality gate
- Delivery Engineer: connectors, data pipeline, automation
- Ops Analyst: briefing, validation, user adoption

Client:
- Executive Sponsor: priorities and escalation decisions
- Process Owner(s): sales/procurement/quality/ops inputs
- Data Steward: source hygiene and access continuity

## Required Templates (Per Client)

- `daily_ops_update`
- `quality_incident_log`
- `procurement_supplier_tracker`
- `sales_market_signal`
- lead tracker (for commercial funnel if requested)

## Quality Gates

Must pass before handover:
- data coverage score >= 80
- open high-priority items all have owner + due date
- weekly brief generated from live data (not manual copy)
- no secrets stored in repo artifacts

## KPIs to Track

Core:
- open high-priority tracker count
- incident-to-CAPA closure cycle
- supplier response SLA delay
- overdue payment follow-up cycle
- weekly action completion rate

Commercial:
- lead-to-proposal cycle time
- proposal-to-win rate
- implementation time to first measurable value

## Multi-Client Repeatability Rules

- always start from standard templates, then customize
- keep connector layer reusable; isolate client credentials
- keep role control towers consistent across clients
- add new module only after baseline loop is stable
