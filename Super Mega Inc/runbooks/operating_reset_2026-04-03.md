# SuperMega Operating Reset

Date: 2026-04-03

## What SuperMega Is

SuperMega should be a small workflow software company, not a broad AI category company.

The public company should sell three offers only:

1. Lead Finder
   Search a market, keep the right leads, and create the first follow-up.
2. Action OS Starter
   Bring a lead list or messy team updates and run one queue with owners and due dates.
3. Myanmar Ops Control
   A managed operating layer for receiving, supplier issues, stock pressure, and payment follow-up without a full ERP rollout.

Everything else stays internal until it is equally clean.

## What Stays Internal

- Private sales cockpit
- Solution Architect
- Director dashboard
- Approval queue
- Exception queue
- R&D or lab surfaces
- Generic agent platform language
- Multi-pack product catalog

These are delivery tools, not front-door offers.

## Why Earlier Work Felt Fake

1. The site described software instead of closing one working loop.
2. Too many public product names competed with each other.
3. The public shell exposed auth, storage, and workspace mechanics before users got value.
4. Internal packaging leaked into public UX.
5. The company story was ahead of the actual live deployment path.

## What Makes Lead Finder Better Than Google or Facebook Search

Google and Facebook help find pages. Lead Finder should help decide what to do next.

Lead Finder advantage:

- turn search into a shortlist
- keep only leads worth chasing
- create first outreach
- create first follow-up task
- save the result into one shared queue

If it does not close that loop, it is not better than ordinary search.

## The Agent Company Model

Agents should exist only where they create concrete operating records.

### Revenue Scout

- run saved lead hunts
- score results
- create outreach drafts
- create first follow-up tasks

Output:
- saved leads
- outreach drafts
- queue items

### Onboarding Architect

- take pasted client data
- map it into the first working queue
- recommend the first template set

Output:
- first workspace
- first queue
- onboarding checklist

### Ops Triage

- turn messy daily updates into queue items
- identify blockers, owners, and due dates

Output:
- queue items
- escalation items

### Director Brief

- summarize the real state from the queue
- show top risks and top follow-ups

Output:
- one short daily brief

## Product Mechanisms That Matter

These are the durable loops worth building:

1. Lead hunt loop
   Saved searches produce new leads.
2. First follow-up loop
   Every saved lead creates one next action.
3. Queue loop
   Every update becomes an owner and due date.
4. Escalation loop
   Aging items and blocked items become exceptions.
5. Director brief loop
   The queue produces one short management summary.

## What To Build Next

### Must do next

1. Shared public workspace as the default save path
2. Recoverable onboarding with company and work email
3. One live backend host for public save and app routes
4. Billing, pricing, onboarding, support, telemetry

### Build after that

1. Myanmar Ops Control
2. Director Brief
3. Follow-up Reply Assistant

### Do not prioritize

1. public AI-agent taxonomy
2. broad ERP suite marketing
3. many public product tabs
4. internal modules as public offers

## Execution Standard

A new feature is only real if:

- a user can understand it in one screen
- a user can do one job without setup confusion
- the system creates a durable record
- the next action is clearer after the feature than before

If those four conditions are not met, it is not ready.
